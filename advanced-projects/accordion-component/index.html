‚ùì What is an Accordion Component? Popular
‚ñº
An accordion component is a UI pattern that allows users to toggle between hiding and showing content sections. It's commonly used for FAQs, menus, and content organization. The name comes from the musical instrument, which expands and contracts similar to how these panels work.

Key Benefits:

  • Saves screen space by hiding content
  • Improves content organization
  • Enhances user experience with progressive disclosure
  • Works great on mobile devices

‚ñº

Basic Structure:

<div class="accordion-item">
  <div class="accordion-header" onclick="toggle()">
    Title
  </div>
  <div class="accordion-content">
    Content here
  </div>
</div>

JavaScript Toggle Function:

function toggle(header) {
  header.classList.toggle('active');
  const content = header.nextElementSibling;
  content.classList.toggle('active');
}
üìö What Are Best Practices?
‚ñº
  1. Use Clear Labels: Make headers descriptive and scannable
  2. Limit Nesting: Avoid more than 2 levels of nested accordions
  3. Provide Visual Feedback: Use icons to indicate expand/collapse state
  4. Consider Mobile: Ensure touch targets are large enough (44x44px minimum)
  5. Performance: Use CSS transitions instead of JavaScript animations
  6. Accessibility: Add ARIA attributes and keyboard support
üéØ When Should I Use Accordions?
‚ñº

Great for:

  • FAQ sections - Perfect for Q&A format
  • Mobile navigation - Saves vertical space
  • Settings panels - Organize related options
  • Product specifications - Group technical details
  • Step-by-step instructions - Break down complex processes

Avoid for:

  • Content users need to compare side-by-side
  • Very short content (just show it instead)
  • Critical information that must be visible
  • Long-form reading (use pagination instead)
